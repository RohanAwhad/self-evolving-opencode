# Test Plans

## Philosophy

- **No mocks.** All test data is real: SQLite fixture DBs, real Redis, pre-recorded LLM responses, real embeddings.
- **Fixtures** = real data snapshots (SQLite files, JSON cassettes, numpy arrays).
- **Skip gracefully** when infra is unavailable (Redis down, no GCP auth).

## Module order (easiest -> hardest, fewest deps first)

---

### 1. `src/llm/cache.py` -- pure Redis ops

**Unit tests:**

| Function | Cases |
|---|---|
| `cache_key` | same inputs -> same key; different inputs -> different key; ordering doesn't matter (sort_keys); non-serializable falls back to `str` |
| `cache_get` | key exists -> returns value; key missing -> `None`; Redis down -> `None` (not crash) |
| `cache_set` | stores value retrievable by `cache_get`; with TTL; without TTL; Redis down -> no-op (not crash) |
| `get_redis` | returns `aioredis.Redis` with correct host/port |

**Fixture:** real Redis on `localhost:6380` (from `compose.yaml`). `pytest.mark.skipif` if unavailable. Flush test keys in teardown.

### 2. `src/opencode_db.py` -- pure SQLite

**Fixture:** create a small SQLite DB file with the real schema (`session`, `message`, `part` tables) and seed with ~5 sessions, ~20 messages, ~40 parts. Fixture lives as a conftest factory that creates a temp DB per test.

**Unit tests:**

| Function | Cases |
|---|---|
| `get_sessions` | returns all sessions ordered by `time_updated` DESC; `limit` param works; session with 0 messages (LEFT JOIN); model JSON parsing (valid JSON, malformed JSON -> `"unknown"`); cost/tokens nullable |
| `get_messages_for_session` | correct ordering by `time_created`; text parts extracted; tool parts formatted as `[tool: name]`; session with no messages -> empty list; message with no parts -> empty content |
| `get_conversation_transcript` | output format matches `--- Message N (role) ---`; multi-part messages concatenated |
| `parse_message_range` | `"msgs 1-8"` -> `(0, 8)`; `"msgs 3-3"` -> `(2, 3)`; single number `"5"` -> `(4, 5)`; garbage `"foo"` -> `(0, 999999)`; empty string -> `(0, 999999)` |
| `slice_messages` | correct slice for various ranges; empty input list; range exceeding list length |

### 3. `src/goal_clusterer.py` -- pure compute (stdlib + ML libs)

**Fixture:** list of ~30 goal strings with known semantic groupings (e.g., 10 about "refactoring", 10 about "testing", 10 about "deployment").

**Unit tests:**

| Function | Cases |
|---|---|
| `_embed` | returns `ndarray` with shape `(n, dim)`; same input -> same embedding |
| `_nearest_cluster` | returns correct cluster ID for a point near a known centroid |
| `_merge_small` | small cluster absorbed into nearest large one; all clusters small -> collapse to 0 |
| `_split_large` | cluster > `max_cluster_size` gets sub-clustered; result clusters all <= max_size |
| `_renumber` | `[3, 7, 3, 12]` -> `[0, 1, 0, 2]`; already contiguous -> no change |
| `cluster_goals` | end-to-end: known groupings cluster together; single goal -> 1 cluster; 2 goals -> 1 cluster (below min_cluster_size); all identical goals -> 1 cluster |

## Test infrastructure

| Item | Detail |
|---|---|
| Framework | `pytest` + `pytest-asyncio` |
| Layout | `tests/` dir, one file per module: `test_cache.py`, `test_opencode_db.py`, `test_goal_clusterer.py`, `test_llm.py`, `test_goal_checker.py`, `test_goal_extractor.py`, `test_play.py` |
| Shared fixtures | `tests/conftest.py` -- SQLite DB factory, Redis connection (skip if down), cache pre-seeder |
| Fixture data | `tests/fixtures/` -- seed data JSON files for DB rows, pre-recorded LLM responses |
| Markers | `pytest.mark.redis` (needs Redis), `pytest.mark.live` (hits real LLM API) |
| Default run | `uv run pytest` runs everything except `live`-marked tests |

## Implementation order

1. Add `pytest` + `pytest-asyncio` to dev deps
2. `tests/conftest.py` -- DB factory fixture + Redis fixture
3. `test_opencode_db.py` -- highest value, zero external deps, most edge cases
4. `test_cache.py` -- simple, validates Redis degradation
5. `test_goal_clusterer.py` -- pure compute, interesting edge cases
6. `test_llm.py` -- `strip_fences` + cache-hit paths
7. `test_play.py` -- formatters first, CLI integration after
8. `test_goal_checker.py` + `test_goal_extractor.py` -- last, most integration-y
