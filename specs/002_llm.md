# 002 — LLM Layer (`src/llm/`)

## Purpose

Thin wrapper around `AsyncAnthropicVertex`. All LLM calls go through `complete()` or `complete_tool()`. Both cache results in Redis and auto-retry on failure.

## Model

Default: `claude-opus-4-6@default` via Vertex AI. Env vars: `ANTHROPIC_VERTEX_PROJECT_ID`, `GOOGLE_VERTEX_LOCATION`.

## API

### `complete(messages, *, model=DEFAULT_MODEL, max_tokens=4096, system=None, **kwargs) → str`
Text completion. Returns raw response string.

### `complete_tool(messages, *, tool, model=DEFAULT_MODEL, max_tokens=4096, system=None, **kwargs) → dict`
Forced tool-use completion. Returns the parsed tool input dict.

### `cache_key(**kwargs) → str`
SHA-256 hash of sorted kwargs → deterministic cache key.

### `cache_get(key) → str | None`
Redis GET, returns `None` if Redis is down (no error).

### `cache_set(key, value, ttl=86400) → None`
Redis SETEX, no-op if Redis is down (no error).

## Caching (`cache.py`)

- Redis on `localhost:6380` (env: `REDIS_HOST`, `REDIS_PORT`)
- Keys are SHA-256 hashes of all call kwargs (deterministic)
- TTL: 86400s (1 day) default
- Silent degradation — if Redis is unavailable, LLM calls proceed uncached

## Retry (`llm/__init__.py`)

- 5 retries with exponential backoff + jitter (via tenacity)
- Handles transient API failures, rate limits, network issues

## Internal

### `get_client(project_id, location) → AsyncAnthropicVertex`
Creates a Vertex AI client. Uses env vars for defaults.

### `_build_messages(messages, system) → list`
Prepends system prompt if provided. Handles `system` kwarg for Anthropic API.

## Testing (`tests/test_llm.py` — 24 tests)

**Pure unit tests** (no external deps):
- `strip_fences`: markdown code fences removal, no fences, nested fences, only opening fence
- `get_client`: returns `AsyncAnthropicVertex`, custom project/region handling
- `cache_key` integration: `complete` and `complete_tool` produce deterministic keys for same inputs

**Integration tests** (`@pytest.mark.redis` — require Redis on localhost:6380):
- `complete` cache hit: returns cached value without LLM call
- `complete` cache miss: calls LLM, stores result in Redis
- `complete_tool` cache hit: returns cached dict
- `complete_tool` cache miss: calls LLM, returns tool input dict

**Live tests** (`@pytest.mark.live` — skipped by default):
- Hits real Vertex AI API for `complete()` and `complete_tool()`
