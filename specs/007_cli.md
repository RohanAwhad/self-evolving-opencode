# 007 — CLI Entrypoint (`play.py`)

## Purpose

Main CLI for listing sessions, extracting goals, checking achievements, summarizing threads, and clustering.

## Modes

### 1. List Sessions (default)
```bash
uv run python play.py [-n LIMIT] [--dir DIR] [--agent AGENT]
```
Fetches sessions, applies filters, prints formatted table with columns: `#`, `Updated`, `Agent`, `Model`, `Msgs`, `Cost`, `In`, `Out`, `Title`.

### 2. Extract Goals
```bash
uv run python play.py --goals SESSION_ID
uv run python play.py --goals ROW_INDEX   # from listing table
```
Prints numbered goals with `[message_range] title` and description.

### 3. Check Goals
```bash
uv run python play.py --goals SESSION_ID --check
```
Also runs `check_goal_achieved()` on each goal's message slice. Prints `ACHIEVED`/`NOT ACHIEVED` verdicts with reasoning.

### 4. Summarize
```bash
uv run python play.py --goals SESSION_ID --summarize
```
Also runs `summarize_conversation()` on each goal's thread. Prints structured markdown.

### 5. Batch + Cluster
```bash
uv run python play.py --goals-file sessions.txt --cluster [--summarize]
```
Extracts goals from all session IDs in file, clusters them, prints each cluster with goal list. Optionally summarizes each thread within cluster output.

## Flags

| Flag | Description |
|---|---|
| `-n`, `--limit` | Max sessions to show (default: 30) |
| `--dir` | Filter by directory substring |
| `--agent` | Filter by agent name |
| `--goals` | Session ID or row index for goal extraction |
| `--goals-file` | File with one session ID per line |
| `--check` | With `--goals`: check goal achievement |
| `--cluster` | With `--goals-file`: cluster extracted goals |
| `--summarize` | Also summarize each thread |
| `--min-cluster-size` | Minimum cluster size (default: 5) |
| `--max-cluster-size` | Maximum cluster size (default: 100) |

## Index Resolution

If `--goals` receives a numeric value ≤ 500, it's treated as a row index into the filtered session listing. Resolved to a real session ID before processing.

## Helpers

### `fmt_time(ts) → str`
Formats epoch ms, epoch s, or ISO string to local `YYYY-MM-DD HH:MM`.

### `fmt_cost(cost) → str`
`$X.XX` format, 4 decimal places if < $0.01.

### `fmt_tokens(n) → str`
Human-readable: `1.5M`, `12.3K`, or raw number.

### `process_goals(session_id, check=False, summarize=False) → None`
Orchestrates extraction + optional checking + optional summarization for a single session.

### `summarize_thread(session_id, goal) → str`
Fetches rich messages for the goal's message range and summarizes them.

## Testing

**No dedicated test file** — CLI integration is exercised indirectly through:
- Module-level tests for each underlying component (DB, LLM, extraction, checking, summarization, clustering)
- Manual smoke tests documented in `.test_plans.md`

**Formatting helpers** tested via manual verification:
- `fmt_time`: epoch ms, epoch s, ISO string, `0`/`None`/`""` → `"n/a"`
- `fmt_cost`: `< $0.01` → 4 decimal places, `≥ $0.01` → 2 decimal places
- `fmt_tokens`: millions (`1.5M`), thousands (`12.3K`), raw (`500`)
